# CLAUDE.md

This file is the entry point for all Claude Code sessions.

Read the following files in order before writing any code:

1. `CLAUDE_DESIGN.md` — design identity, emotional registers, motion language, spatial system, typography
2. `CLAUDE_DOCTRINE.md` — engineering principles, scope rules, decision framework
3. `CLAUDE_TECHNICAL.md` — schema, file structure, auth strategy, patterns

All three are required reading. Design establishes *what it feels like*. Doctrine establishes *why*. Technical establishes *how*.

When they conflict: Design and Doctrine take precedence over Technical. Doctrine takes precedence over Design on engineering decisions.

## When Uncertain

If a decision is ambiguous — scope, architecture, dependency choice, data model change, or visual direction — stop and surface the question rather than making an assumption.

Small decisions compound. It is better to ask once than to refactor later.
